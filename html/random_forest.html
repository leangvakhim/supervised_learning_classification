The Wisdom of the Crowd

You already know how a single Decision Tree works. It asks a series of questions to make a prediction. However, a single tree can sometimes be too rigid, overfitting the data it trained on and making poor predictions on new data.

A Random Forest is an ensemble method. Instead of relying on one tree, it builds a "forest" of many different decision trees and combines their predictions. It's like asking a whole classroom of students for an answer rather than just one.

Many trees working together

Student Example: Predicting Final Grades

Imagine we want to predict if a student will pass or fail a difficult exam.

The Data

We have data on past students: hours studied, attendance rate, assignment scores, and whether they passed.

Step 1: Bootstrapping (Creating the Data)

If every tree used the exact same data, they would all be identical. Random Forest prevents this using Bootstrap Aggregating (Bagging).

For each tree, we create a new training dataset by randomly selecting samples from the original data with replacement.

Original Data
[A, B, C, D, E]
→
Tree 1 Data: [A, B, B, D, E]
Tree 2 Data: [A, C, C, C, E]
Tree n Data: [B, D, E, E, E]

Notice how some students (like C) might appear multiple times in one set, and not at all in another.

Step 2: Feature Randomness

To make the trees even more diverse, Random Forest adds another layer of randomness during the growth of each tree.

At each node split in the tree, instead of looking at all features (e.g., Study Hours, Attendance, Past Grades, Sleep), it only considers a random subset of features.

Standard formula for subset size \( m \):

$$ m \approx \sqrt{p} $$

Where \( p \) is the total number of features (e.g., if we have 9 features about a student, each node split only looks at roughly 3 random features to decide how to split).

Step 3: Aggregation (Making the Prediction)

Once all the trees are built, how do we get the final answer? It depends on the task!

Classification (Pass/Fail)

Majority Voting: The class predicted by the most trees wins.

$$ \hat{y} = \text{mode}\{h_1(x), h_2(x), ..., h_B(x)\} $$

Where \( h_b(x) \) is the prediction of the \( b \)-th tree, and \( B \) is the total number of trees.

Regression (Predicting Score)

Averaging: The final prediction is the average of all tree predictions.

$$ \hat{y} = \frac{1}{B} \sum_{b=1}^{B} h_b(x) $$

We simply add up all the predicted exam scores from each tree and divide by the number of trees.
